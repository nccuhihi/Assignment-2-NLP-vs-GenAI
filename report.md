# 比較分析報告


## 主題： 傳統 NLP vs 現代 AI：文本處理方法實作比較

### 1. 比較分析報告 (Comparison Analysis)

本報告基於 comparison.py 的效能評測結果，針對「相似度計算」、「文本分類」與「自動摘要」三項任務，對比傳統方法（TF-IDF/規則式）與現代 GenAI 方法（Gemini API）的各項指標。

| 任務類型 | 評估指標 | 傳統 NLP 方法 (TF-IDF / Rule-based) | 現代 AI 方法 (GenAI / LLM) | 差異分析 |
| :--- | :--- | :--- | :--- | :--- |
| **相似度計算** | 處理時間 | **極快 (< 0.001 秒)** | 較慢 (~ 0.5 - 1.0 秒) | 傳統方法僅涉及矩陣運算，速度極快；AI 需等待網路傳輸與模型推論。 |
| **相似度計算** | 準確率/合理性 | 高 (數學精確) | **極高 (語意理解)** | 傳統方法依賴字面重疊，無法識別同義詞；AI 能理解語意關聯。 |
| **相似度計算** | 成本 | **極低 (本地運算)** | 中 (API 呼叫費用) | 大量數據下，API 成本不可忽視。 |
| **文本分類** | 處理時間 | **極快 (< 0.001 秒)** | 較慢 (~ 1.0 - 1.5 秒) | 規則判斷近乎即時；AI 生成回應需要時間。 |
| **文本分類** | 準確率 | 中 (依賴關鍵字庫) | **極高 (Zero-shot)** | 規則式容易被否定詞或複雜句型誤導；AI 具備邏輯推理能力。 |
| **文本分類** | 擴充彈性 | 低 (需人工維護詞庫) | **高 (任意指定 Prompt)** | AI 可透過修改 Prompt 即時支援新類別，無需重新編寫程式碼。 |
| **自動摘要** | 處理時間 | **快 (< 0.01 秒)** | 慢 (~ 2.0 - 4.0 秒) | 統計式摘要僅需計算詞頻與排序。 |
| **自動摘要** | 語句通順度 | 低 (句子拼接) | **高 (生成式改寫)** | 傳統方法產出的摘要較生硬；AI 能寫出流暢且連貫的短文。 |
| **自動摘要** | 方法性質 | 抽取式 (Extractive) | 生成式 (Abstractive) | 傳統方法保留原文句子；AI 能融會貫通後重新表述。 |


### 2. 心得與反思 (Reflection)
**(1) 方法特性比較：精確匹配與語意理解的權衡**

透過本次作業對比傳統 TF-IDF 演算法與現代 GenAI 模型的特性，最顯著的差異在於「精確匹配」與「語意理解」之間的權衡。傳統方法如 TF-IDF 與規則式分類，本質上是基於統計學與明確邏輯的產物，它們的優勢在於極高的執行效率與完全的可解釋性。在計算文本相似度時，TF-IDF 能夠精確地捕捉到共用關鍵詞的權重，對於專有名詞密集的技術文件檢索極為有效，且運算資源需求極低，幾乎可以在本地端即時完成大量數據的運算。然而，其劣勢也相當明顯，即無法處理同義詞或隱喻，一旦查詢詞與文件中的關鍵詞未能在字面上完全匹配，系統便無法識別其關聯性，導致「詞不達意」的檢索遺漏。相比之下，基於大型語言模型的 GenAI 展現了強大的泛化能力與語意理解力。透過 Embeddings 向量化技術，模型不再受限於字面符號，而是能捕捉文字背後的高維度深層意涵，即便兩句話用詞完全不同，模型也能精準判斷其語意相似度。在自動摘要與分類任務中，GenAI 更展現了「融會貫通」的生成能力，能產出流暢且符合人類邏輯的結果，這是傳統統計式摘要僅能進行句子拼接所無法比擬的。然而，這種強大的能力伴隨著高昂的 API 呼叫成本、較高的網路延遲時間，以及偶爾出現的幻覺風險，這使得開發者在選擇方法時，必須在效能、成本與準確度之間進行極為審慎的評估。

**(2) 實作心得：從底層邏輯建構到高層指令溝通**

在實際撰寫程式碼的過程中，最讓我印象深刻的是從「底層邏輯建構」跨越到「高層指令溝通」的巨大體驗落差。在實作 Part A 時，我必須親手處理斷詞、建立詞袋模型，並一步步撰寫 TF 與 IDF 的數學公式。雖然過程較為繁瑣，需要細心處理分母為零、平滑化計算以及矩陣維度對齊等細節，但當我看到程式成功計算出關鍵詞權重並篩選出最具代表性的詞彙時，那種對於演算法內部運作的掌控感是非常踏實的。我深刻理解到每一個相似度分數的由來，也明白了為什麼某些長句會因為正規化而被降低權重，這種「白箱」的開發體驗對於理解 NLP 基礎至關重要。反觀 Part B 的實作過程則完全不同，原本需要數十行程式碼才能完成的分類或摘要任務，現在僅需設計好一個精準的 Prompt，呼叫一次 API 就能得到甚至超越預期的結果。這種「Zero-shot」的零樣本學習能力讓我感到驚艷，不需要蒐集大量標註資料訓練模型，就能直接處理複雜的自然語言任務。然而，這種便利性也帶來了一種不確定感，因為我無法像在 Part A 那樣精確控制模型輸出的每一個字，有時為了讓模型輸出特定格式（如精確的信心分數格式），需要反覆調整提示詞的語氣、限制條件與範例，這讓我體會到在現代 AI 開發中，「提示工程」已成為取代傳統程式邏輯控制的關鍵技能，我們從單純的程式設計者轉變成了 AI 的引導者與溝通者。

**(3) 應用建議：打造優勢互補的混合式系統**

綜合上述的特性比較與實作經驗，我認為在未來的實際應用場景中，不應存在「完全取代」的思維，而應採取「優勢互補」的混合策略。在資源受限的邊緣運算裝置、即時性要求極高的關鍵字過濾系統，或是對於金融、法律等需要百分之百可解釋性的領域，傳統的 TF-IDF 與規則式方法依然是最佳選擇。它們不僅穩定、透明，且不會產生資安外洩的疑慮，能以最低的成本解決單純的檢索問題。然而，對於需要高度使用者互動、複雜語意理解或內容生成的應用，如智慧客服、語意搜尋或個人助理，GenAI 則是不可或缺的核心。最理想的系統架構應是導入「檢索增強生成」（RAG）的概念，結合兩者的長處。我們可以先利用 TF-IDF 或 BM25 演算法進行第一階段的快速檢索，從海量數據中篩選出相關性最高的候選文檔，這一步驟能大幅降低運算成本並確保檢索範圍的精準度；接著，再利用 GenAI 的語意分析能力對這些候選文檔進行第二階段的精確排序與閱讀理解，最終生成流暢的回答。這種混合架構不僅解決了 GenAI 處理長文本時的 Token 限制與高昂成本問題，同時也利用了傳統方法的高效檢索能力，有效降低模型產生幻覺的風險。因此，身為開發者，我們不應盲目追求最新技術，而應根據場景的具體需求，靈活調配傳統算法與現代模型的比例，打造出既高效又智慧的解決方案。
